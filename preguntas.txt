1. ¿En qué consiste el principio de responsabilidad única? ¿Cuál es su propósito?
El principio de responsabilidad única consiste en que una clase debe tener una y solo una única causa por la cual puede ser modificada. El proposito de este principio es no tener una clase con dos responsabilidades y asi tener el código limpio y organizado.

2. ¿Qué características tiene según tu opinión “buen” código o código limpio?
* Tener clase por responsabilidad.
* Una clase no debe tener muchas lineas, si este las tiene es porque es necesario dividirla en dos clases dado que tiene mas de una responsabilidad.
* Comentarios dando una idea de lo que hacen las funciones.
* No usar más if de los que se debe.
* Dividir las funciones grande en mini funciones para tener claro la funcionalidad de cada cosa.
* Nombrar las variables y funciones tal que permitan deducir que son o que hacen.
